Итак, мы научились делать git diff.
Вариант первый - у нас есть измененные  отслеживаемые файлы в рабочей области.
Возникает простой вопрос - что в них поменалось. Для этого следует сделать git diff чтобы
увидеть изменения в виде патча.

Вторая возможность:
Используется тогда, когда я добавил файлы из рабочей области в отслеживаемое и
уже готов коммитить.
Мне интересно увидеть, что именно войдет в мой коммит и тогда я делаю
 git diff --staged или же git diff --cached # по сути одна и та же команда,
вторая - более универсальный вариант для старых версий гита.
